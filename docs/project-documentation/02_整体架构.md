# Mini-Agent 整体架构

## 1. 架构概览

Mini-Agent 采用**分层架构**设计，遵循**端口与适配器 (Ports and Adapters)** 模式。

## 2. 核心设计原则

### 2.1 分层职责

| 层级 | 职责 | 依赖方向 |
|------|------|---------|
| Surface Layer | 用户交互入口 | → Application |
| Application Layer | 用例编排、端口定义 | → Agent Core |
| Agent Core Layer | Agent 核心逻辑、状态管理 | → Service |
| Service Layer | 业务服务、数据访问 | → Infrastructure |
| Infrastructure Layer | 外部系统集成、持久化 | 无依赖 |

### 2.2 端口与适配器

```
┌─────────────────────────────────────────────────────────────────┐
│                        Application Layer                         │
│  ┌───────────────────────────────────────────────────────────┐  │
│  │                    Port (接口定义)                         │  │
│  │  - AgentRuntimePort                                       │  │
│  │  - ModelRuntimePort                                       │  │
│  │  - WorkspaceRuntimePort                                   │  │
│  │  - SessionTaskPort                                        │  │
│  └───────────────────────────────────────────────────────────┘  │
│                              │                                   │
│                              ▼                                   │
│  ┌───────────────────────────────────────────────────────────┐  │
│  │                  Adapter (适配器实现)                      │  │
│  │  - RuntimeSessionRunControlHandler                        │  │
│  │  - ModelRuntimeAdapter                                    │  │
│  │  - WorkspaceRuntimeAdapter                                │  │
│  └───────────────────────────────────────────────────────────┘  │
└─────────────────────────────────────────────────────────────────┘
```

### 2.3 数据流向

```
用户输入
    │
    ▼
Surface Layer (TUI/Desktop/Gateway)
    │
    ▼
Application Layer (Facades → Use Cases)
    │
    ▼
Agent Core Layer (Engine → Execution → Context)
    │
    ├──────────────────────────────┐
    ▼                              ▼
Service Layer              Infrastructure Layer
(User Services)            (Model Manager, LLM, Tools)
    │                              │
    └──────────────┬───────────────┘
                   ▼
             外部系统 (LLM API, 文件系统, MCP 服务器)
```

## 4. 核心概念

### 4.1 Agent 实体模型

```
┌─────────────────────────────────────────────────────────────────┐
│                        AgentProfile                              │
│  (Agent 静态配置 - 持久化)                                       │
│  - agent_profile_id                                              │
│  - name, description                                             │
│  - default_model_preference                                      │
│  - tool_policy                                                   │
│  - skill_preferences                                             │
└─────────────────────────────────────────────────────────────────┘
                                │
                                │ 1:N
                                ▼
┌─────────────────────────────────────────────────────────────────┐
│                       AgentInstance                              │
│  (Agent 运行时实例 - 有状态)                                     │
│  - agent_instance_id                                             │
│  - agent_profile_id                                              │
│  - lifecycle_state (COLD/READY/RUNNING/WAITING/...)             │
│  - active_run_id                                                 │
│  - current_workspace_id                                          │
│  - current_session_id                                            │
└─────────────────────────────────────────────────────────────────┘
                                │
                                │ 1:N
                                ▼
┌─────────────────────────────────────────────────────────────────┐
│                           Run                                    │
│  (单次执行 - 独立于 Session)                                     │
│  - run_id                                                        │
│  - agent_instance_id                                             │
│  - status (PENDING/RUNNING/COMPLETED/FAILED/...)                │
│  - started_at, completed_at                                      │
│  - checkpoint_head_id                                            │
└─────────────────────────────────────────────────────────────────┘
```

### 4.2 工作空间与附件

```
┌─────────────────────────────────────────────────────────────────┐
│                        Workspace                                 │
│  (工作空间 - 项目边界)                                           │
│  - workspace_id                                                  │
│  - workspace_kind (default/project)                             │
│  - root_path                                                     │
│  - permission_table                                              │
└─────────────────────────────────────────────────────────────────┘
                                │
                                │ 1:N
                                ▼
┌─────────────────────────────────────────────────────────────────┐
│                   WorkspaceAttachment                            │
│  (工作空间附件 - Agent 绑定)                                     │
│  - attachment_id                                                 │
│  - workspace_id                                                  │
│  - agent_profile_id                                              │
│  - bound_at                                                      │
└─────────────────────────────────────────────────────────────────┘
                                │
                                │ 1:N
                                ▼
┌─────────────────────────────────────────────────────────────────┐
│                    SessionAttachment                             │
│  (会话附件 - Run 绑定)                                           │
│  - attachment_id                                                 │
│  - session_id                                                    │
│  - run_id                                                        │
│  - attached_at                                                   │
└─────────────────────────────────────────────────────────────────┘
```

### 4.3 模型绑定

```
┌─────────────────────────────────────────────────────────────────┐
│                      AgentModelPolicy                            │
```

## 5. 运行时架构

### 5.1 Agent 执行循环

```
┌─────────────────────────────────────────────────────────────────┐
│                        AgentEngine                               │
│  ┌───────────────────────────────────────────────────────────┐  │
│  │                      AgentLoop                             │  │
│  │  ┌─────────────────────────────────────────────────────┐  │  │
│  │  │                   Step Loop                          │  │  │
│  │  │  1. Prepare Turn Context                             │  │  │
│  │  │     - ContextAssembler                               │  │  │
│  │  │     - SkillResolver                                  │  │  │
│  │  │     - MemoryResolver                                 │  │  │
│  │  │                                                      │  │  │
│  │  │  2. Call LLM                                         │  │  │
│  │  │     - FailoverLLMClient                              │  │  │
│  │  │     - Stream/Buffered Mode                           │  │  │
│  │  │                                                      │  │  │
│  │  │  3. Process Response                                 │  │  │
│  │  │     - Parse Tool Calls                               │  │  │
│  │  │     - Execute Tools                                  │  │  │
│  │  │     - Update State                                   │  │  │
│  │  │                                                      │  │  │
│  │  │  4. Check Transition                                 │  │  │
│  │  │     - CONTINUE / COMPLETE / FAILED / CANCELLED       │  │  │
│  │  └─────────────────────────────────────────────────────┘  │  │
│  └───────────────────────────────────────────────────────────┘  │
└─────────────────────────────────────────────────────────────────┘
```

### 5.2 工具执行流程

```
Tool Call Request
        │
        ▼
┌───────────────────┐
│ PermissionEngine  │ ──── 评估权限
└───────────────────┘
        │
        ▼
┌───────────────────┐
│   SandboxManager  │ ──── 沙箱隔离 (可选)
└───────────────────┘
        │
        ▼
┌───────────────────┐
│  Tool Execution   │ ──── 执行工具
└───────────────────┘
        │
        ▼
┌───────────────────┐
│ MutationLedger    │ ──── 记录变更 (可选)
└───────────────────┘
        │
        ▼
Tool Result
```

## 6. 数据持久化

### 6.1 存储位置

| 数据类型 | 存储位置 | 格式 |
|---------|---------|------|
| Provider 配置 | `~/.mini-agent/providers.json` | JSON |
| Agent 模型绑定 | `~/.mini-agent/agent_model_binding.json` | JSON |
| 会话记录 | `workspace/.mini-agent/sessions/*.jsonl` | JSONL |
| 检查点 | `workspace/.mini-agent/checkpoints/*.json` | JSON |
| 技能文件 | `./skills/*/SKILL.md` | Markdown |
| MCP 配置 | `./mcp.json` | JSON |

### 6.2 会话存储格式

```jsonl
{"event": "user_message", "content": "...", "timestamp": "..."}
{"event": "assistant_message", "content": "...", "tool_calls": [...]}
{"event": "tool_result", "tool_name": "...", "result": "..."}
{"event": "checkpoint", "checkpoint_id": "...", "state": {...}}
```

## 8. 安全架构

```
┌─────────────────────────────────────────────────────────────────┐

│  │  - 控制文件访问范围                                        │  │
│  └───────────────────────────────────────────────────────────┘  │
│  ┌───────────────────────────────────────────────────────────┐  │
│  │                  OutsideZonePolicy                         │  │

│  │  - 作用域限制                                              │  │

└─────────────────────────────────────────────────────────────────┘
```
