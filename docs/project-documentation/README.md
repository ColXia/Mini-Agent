## 使用说明

### 学习工程开发

建议阅读顺序：
1. 先阅读 [01_项目概览.md](01_项目概览.md) 了解项目背景
2. 再阅读 [02_整体架构.md](02_整体架构.md) 理解系统设计
3. 按需深入各模块文档

### 批注修改需求

1. 在对应文档中添加批注
2. 使用 `[批注]` 或 `[修改建议]` 标记
3. 说明具体问题和期望行为

示例：
```markdown
### 3.1 AgentModelService

[批注] 当前设计缺少模型别名支持，建议添加别名解析层。

期望行为：
- 用户输入 "opus" 自动解析为 "claude-opus-4-6"
- 支持用户自定义别名
```
