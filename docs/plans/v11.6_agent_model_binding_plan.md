## 一、设计概述

### 1.1 核心原则

**Agent 是模型的使用者，模型服务是模型的提供者**

- Agent 侧负责：模型绑定配置、故障转移决策、上下文管理
- 模型侧负责：协议适配、健康监控、熔断器、Token 统计

### 1.2 职责边界

```
┌─────────────────────────────────────────────────────────┐
│                    Agent 侧职责                          │
├─────────────────────────────────────────────────────────┤
│  ✓ 模型绑定配置 (一个 Agent 可绑定多个模型)             │
│  ✓ 故障转移决策 (收到熔断通知后自动切换)               │
│  ✓ 上下文管理 (根据 Token 消耗判断压缩)                 │
│  ✓ 能力适配 (根据模型能力调整行为)                      │
│  ✓ 错误处理 (根据错误类型决定重试/转移/报错)            │
└─────────────────────────────────────────────────────────┘

┌─────────────────────────────────────────────────────────┐
│                    模型侧职责                            │
├─────────────────────────────────────────────────────────┤
│  ✓ Provider 管理 (注册、配置、验证)                     │
│  ✓ 协议适配 (OpenAI/Anthropic)                          │
│  ✓ 参数配置 (temperature, max_tokens 等)                │
│  ✓ 健康监控 (成功率、延迟统计)                          │
│  ✓ 熔断器 (连续失败后熔断，通知 Agent)                  │
│  ✓ Token 统计 (输入/输出/成本)                          │
│  ✓ 能力探测 (手动触发，结果缓存)                        │
│  ✗ 故障转移 (不负责，由 Agent 侧决定)                   │
└─────────────────────────────────────────────────────────┘
```

## 二、Agent 模型绑定设计

### 2.1 绑定结构

**一个 Agent 可以绑定多个模型，配置保存在 Agent 侧**

```python
@dataclass
class AgentModelBinding:
    """Agent 模型绑定配置"""
    agent_id: str
    models: list[ModelBindingEntry]
    failover_enabled: bool = True  # 是否启用故障转移

@dataclass
class ModelBindingEntry:
    """单个模型绑定条目"""
    config_id: str           # 模型配置 ID
    role: str                # primary / fallback_1 / fallback_2 / ...
    custom_name: str | None  # 自定义名称 (覆盖原名)
    enabled: bool = True
```

### 2.2 配置示例

```json
{
  "agent_id": "code-assistant",
  "models": [
    {
      "config_id": "sonnet-code",
      "role": "primary",
      "custom_name": "代码主力",
      "enabled": true
    },
    {
      "config_id": "haiku-fast",
      "role": "fallback_1",
      "custom_name": "快速响应",
      "enabled": true
    },
    {
      "config_id": "opus-power",
      "role": "fallback_2",
      "custom_name": "强力兜底",
      "enabled": true
    }
  ],
  "failover_enabled": true
}
```

### 2.3 绑定管理

**用户从模型池选取模型配置到 Agent**


## 三、故障转移设计

### 3.1 转移流程

```
┌─────────────────────────────────────────────────────────┐
│                    故障转移流程                          │
├─────────────────────────────────────────────────────────┤
│                                                         │
│  1. Agent 调用主模型 (primary)                          │
│     │                                                   │
│     ▼                                                   │
│  2. 模型侧熔断器检测到连续失败                          │
│     │                                                   │
│     ▼                                                   │
│  3. 模型侧触发熔断，通知 Agent                          │
│     │                                                   │
│     ▼                                                   │
│  4. Agent 收到熔断通知                                  │
│     │                                                   │
│     ▼                                                   │
│  5. Agent 判断是否启用故障转移                          │
│     ├── 是 → 切换到 fallback_1                          │
│     └── 否 → 返回错误                                   │
│     │                                                   │
│     ▼                                                   │
│  6. 调用备用模型                                        │
│     ├── 成功 → 继续工作                                 │
│     └── 失败 → 尝试下一个备用                           │
│     │                                                   │
│     ▼                                                   │
│  7. 全部备用模型失败                                     │
│     │                                                   │
│     ▼                                                   │
│  8. 返回"模型服务不可用"错误                            │
│                                                         │
└─────────────────────────────────────────────────────────┘
```

### 3.2 转移触发条件

| 触发条件 | Agent 处理 |
|---------|-----------|
| 收到熔断通知 | 自动切换到备用模型 |
| 调用超时 (可重试) | 先重试，失败后转移 |
| 连接不稳定 | 可能转移 |
| 认证失败 | 不转移，报错 |
| 配额耗尽 | 可能转移 |
| 模型不支持某能力 | 不转移，报错 |

### 3.3 转移策略配置

```python
@dataclass
class FailoverPolicy:
    """故障转移策略"""
    enabled: bool = True
    retry_before_failover: bool = True  # 转移前是否先重试
    max_retries: int = 2                # 最大重试次数
    failover_order: list[str] = []      # 转移顺序 ["fallback_1", "fallback_2"]
    notify_user_on_failover: bool = True  # 转移时是否通知用户
```

## 五、ModelServicePort 接口定义

**Agent 通过此接口访问模型服务**

```python
class ModelServicePort(Protocol):
    """Agent 访问模型服务的接口"""
    
    # === 绑定与客户端 ===
    def get_binding(self, agent_id: str) -> ModelBindingView:
        """获取 Agent 当前模型绑定"""
    
    def get_client(self, binding: ModelBindingView) -> LLMClient:
        """获取已配置的 LLM 客户端"""
    
    def switch_model(self, agent_id: str, target: str) -> ModelBindingView:
        """切换模型 (支持别名/自定义名)"""
    
    def list_available_models(self) -> list[ModelOptionView]:
        """列出可用模型选项"""
    
    # === 能力查询 ===
    def get_capabilities(self, binding: ModelBindingView) -> ModelCapabilitiesView:
        """获取模型能力"""
    
    # === 状态查询 ===
    def get_health_status(self, model_config_id: str) -> HealthStatusView:
        """获取模型健康状态"""
    
    def get_breaker_status(self, model_config_id: str) -> BreakerStatusView:
        """获取模型熔断状态"""
    
    # === 错误信息 ===
    def get_last_error(self, model_config_id: str) -> ErrorDetail | None:
        """获取最近一次错误详情"""
    
    # === Token 消耗 ===
    def get_token_usage(self, model_config_id: str) -> TokenUsageView:
        """获取 Token 消耗情况"""
    
    # === 回调注册 ===
    def on_breaker_triggered(self, callback: Callable[[str], None]) -> None:
        """注册熔断触发回调"""
    
    def on_health_changed(self, callback: Callable[[str], None]) -> None:
        """注册健康状态变化回调"""
```

## 六、上下文管理机制

### 6.1 职责说明

**Agent 侧负责上下文管理机制的实现**：
- 根据实际工作情况判断是否触发压缩
- 决定压缩策略（保留哪些消息、压缩目标）
- 执行压缩操作

**模型侧只提供数据支持**：
- Token 消耗统计（用于成本控制）
- 上下文窗口大小信息（用于能力展示）

### 6.2 触发条件

Agent 根据 Token 消耗判断是否需要压缩上下文：

```python
def should_compress_context(usage: TokenUsageView) -> bool:
    """判断是否需要压缩上下文"""
    # 使用比例超过阈值
    if usage.usage_ratio > 0.8:
        return True
    # 剩余 Token 不足
    if usage.remaining_tokens < 2000:
        return True
    return False
```

### 6.3 压缩策略配置

```python
@dataclass
class CompressionPolicy:
    """上下文压缩策略"""
    
    # 触发条件
    trigger_ratio: float = 0.8          # 触发压缩的使用比例
    min_remaining_tokens: int = 2000    # 最小剩余 Token
    
    # 压缩目标
    compression_target: float = 0.5     # 压缩目标比例 (压缩到窗口的 50%)
    
    # 消息保留策略
    preserve_system_messages: bool = True   # 保留系统消息
    preserve_recent_messages: int = 3       # 保留最近 N 条消息
    preserve_tool_results: bool = True      # 保留工具调用结果
    
    # 压缩方式
    compression_method: str = "summarize"   # "summarize" / "truncate" / "hybrid"
```

### 6.4 压缩流程

```
┌─────────────────────────────────────────────────────────┐
│                    上下文压缩流程                        │
├─────────────────────────────────────────────────────────┤
│                                                         │
│  1. 获取当前 Token 使用情况                             │
│     │                                                   │
│     ▼                                                   │
│  2. 判断是否需要压缩                                    │
│     ├── 使用比例 > 80% → 需要                          │
│     ├── 剩余 < 2000 → 需要                             │
│     └── 否则 → 不需要                                  │
│     │                                                   │
│     ▼                                                   │
│  3. 确定压缩策略                                        │
│     ├── 计算目标 Token 数                              │
│     └── 确定保留哪些消息                               │
│     │                                                   │
│     ▼                                                   │
│  4. 执行压缩                                            │
│     ├── summarize: 调用模型总结历史消息                 │
│     ├── truncate: 直接截断旧消息                       │
│     └── hybrid: 重要消息保留，其他总结                 │
│     │                                                   │
│     ▼                                                   │
│  5. 更新上下文                                          │
│     │                                                   │
│     ▼                                                   │
│  6. 继续工作                                            │
│                                                         │
└─────────────────────────────────────────────────────────┘
```

### 6.5 压缩实现示例

```python
class ContextCompressor:
    """上下文压缩器"""
    
    def __init__(
        self,
        model_service: ModelServicePort,
        policy: CompressionPolicy,
    ):
        self.model_service = model_service
        self.policy = policy
    
    async def compress_if_needed(
        self,
        messages: list[dict],
        model_config_id: str,
    ) -> list[dict]:
        """如果需要则压缩上下文"""
        
        # 1. 获取 Token 使用情况
        usage = self.model_service.status.get_token_usage(model_config_id)
        
        # 2. 判断是否需要压缩
        if not self._should_compress(usage):
            return messages
        
        # 3. 计算目标 Token 数
        target_tokens = int(usage.context_window * self.policy.compression_target)
        
        # 4. 执行压缩
        if self.policy.compression_method == "summarize":
            return await self._summarize_compress(messages, target_tokens)
        elif self.policy.compression_method == "truncate":
            return self._truncate_compress(messages, target_tokens)
        else:
            return await self._hybrid_compress(messages, target_tokens)
    
    def _should_compress(self, usage: TokenUsageView) -> bool:
        """判断是否需要压缩"""
        if usage.usage_ratio > self.policy.trigger_ratio:
            return True
        if usage.remaining_tokens < self.policy.min_remaining_tokens:
            return True
        return False
    
    async def _summarize_compress(
        self,
        messages: list[dict],
        target_tokens: int,
    ) -> list[dict]:
        """总结式压缩"""
        # 保留系统消息和最近消息
        preserved = self._get_preserved_messages(messages)
        to_summarize = self._get_messages_to_summarize(messages, preserved)
        
        # 调用模型生成总结
        summary = await self._generate_summary(to_summarize)
        
        # 构建新的消息列表
        return [
            *preserved["system"],
            {"role": "assistant", "content": f"[历史对话总结]\n{summary}"},
            *preserved["recent"],
        ]
    
    def _truncate_compress(
        self,
        messages: list[dict],
        target_tokens: int,
    ) -> list[dict]:
        """截断式压缩"""
        preserved = self._get_preserved_messages(messages)
        # 直接返回保留的消息
        return [
            *preserved["system"],
            *preserved["recent"],
        ]
```
